### 2. Service provider owns DI assembly — DONE (b10)

## Base Domain Event Refactor (per-service base events) — separate workstream
This is a separate effort from the PR 841 review items above. It adds a per-module base domain event that owns the shared service injection so concrete events stop repeating the `# * attribute` / `# * init` boilerplate. Each base event extends `DomainEvent`, declares the shared service attribute, and provides the common constructor; concrete events extend the base and keep only their `execute` (and any `@DomainEvent.parameters_required` decorators).
### Completed on `v2.0b10-proto`
- **Pre-step (committed):** the pending `tiferet/events/settings.py` `DomainEvent` parameter-validation + sync/async middleware changes were committed on their own.
- **Phase 1 — `tiferet/events/error.py`:** added `ErrorEvent(DomainEvent)` holding `error_service: ErrorService`; `AddError`, `GetError`, `ListErrors`, `RenameError`, `SetErrorMessage`, `RemoveErrorMessage`, and `RemoveError` extend it.
- **Phase 2 — rename + `tiferet/events/feature.py`:** renamed the domain object `FeatureEvent` → `EventFeatureStep` (it extends `FeatureStep`) to free the `FeatureEvent` name, including mappers `FeatureEventAggregate` → `EventFeatureStepAggregate` and `FeatureEventYamlObject` → `EventFeatureStepYamlObject`. Added `FeatureEvent(DomainEvent)` holding `feature_service: FeatureService`; `AddFeature`, `GetFeature`, `ListFeatures`, `RemoveFeature`, `UpdateFeature`, `AddFeatureStep`, `UpdateFeatureStep`, `RemoveFeatureStep`, and `ReorderFeatureStep` extend it.
- **Rename blast radius:** `tiferet/domain/feature.py`, `tiferet/domain/__init__.py`, `tiferet/mappers/feature.py`, `tiferet/mappers/__init__.py`, `tiferet/contexts/feature.py`, `tiferet/assets/feature.py`, and `tests/{domain,mappers,contexts}/test_feature.py`. Snake_case `feature_event` local/param names were intentionally left unchanged.
### Phase 3 — completed on `v2.0b10-proto`

For each module, the base event was added directly under `# *** events`, every concrete event now extends it, and the per-event `# * attribute` + `# * init` blocks were removed (retaining `execute` and `@DomainEvent.parameters_required`).
- **`tiferet/events/app.py`:** `AppEvent(DomainEvent)` holding `app_service: AppService` → `AddAppInterface`, `GetAppInterface`, `UpdateAppInterface`, `SetAppConstants`, `ListAppInterfaces`, `SetServiceDependency`, `RemoveServiceDependency`, `RemoveAppInterface`. - **Out of scope (unchanged):** `tiferet/events/static.py` (static events, no injected service) and `tiferet/events/settings.py` (defines `DomainEvent`/`AsyncDomainEvent`).

### Pattern reference
```python
# ** event: <domain>_event
class <Domain>Event(DomainEvent):
    '''
    Base event providing the shared <Service> dependency for <domain> domain events.
    '''

    # * attribute: <service>
    <service>: <Service>

    # * init
    def __init__(self, <service>: <Service>):
        # Set the <domain> service dependency.
        self.<service> = <service>
```
