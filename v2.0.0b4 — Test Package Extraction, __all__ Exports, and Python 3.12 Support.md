# v2.0.0b4 — Test Package Extraction, `__all__` Exports, and Python 3.12 Support
**Project:** Tiferet Framework
**Repository:** [https://github\.com/greatstrength/tiferet](https://github.com/greatstrength/tiferet)
**Date:** May 16, 2026
**Target Version:** 2.0.0b4
## 1. Overview
This release restructures the Tiferet repository to mirror the three-project separation established in `tiferet.net` (`Tiferet` / `Tiferet.Testing` / `Tiferet.Tests`), adds explicit `__all__` export lists to every package, and declares official Python 3.10–3.12 support. The goal is to cleanly separate the publishable framework, the publishable test harness, and the framework's own tests — enabling consumers to `pip install tiferet-testing` for harness classes without pulling in 45+ test files.
## 2. .NET Reference Pattern
```warp-runnable-command
tiferet.net/
├── Tiferet/              → Core framework (NuGet: Tiferet, IsPackable=true)
├── Tiferet.Testing/      → Test harnesses (NuGet: Tiferet.Testing, IsPackable=true)
│   └── DomainEventHarness.cs
└── tests/
    ├── Tiferet.Tests/            → Unit tests (IsPackable=false, refs Tiferet + Tiferet.Testing)
    └── Tiferet.Tests.Integration/ → Integration tests (IsPackable=false)
```
Key: `Tiferet.Testing` is a **separate publishable package** containing only test harness base classes. `Tiferet.Tests` references both `Tiferet` and `Tiferet.Testing` but is never published. Shared versioning is managed via `Directory.Build.props`.
## 3. Current Python State
* Test harnesses live **inside the framework package**: `tiferet/mappers/tests/settings.py` (MapperAssertions, AggregateTestBase, TransferObjectTestBase) and `tiferet/events/tests/settings.py` (DomainEventTestBase, ServiceEventTestBase).
* Conftest hooks also live inside: `tiferet/mappers/tests/conftest.py` and `tiferet/events/tests/conftest.py`.
* All 45+ unit test files are co-located in `tiferet/<module>/tests/` — these are **included in the published package** since the `pyproject.toml` find config doesn't exclude them.
* Integration tests live at `tiferet/tests_int/` — also included in the published package.
* No `__all__` on any `__init__.py`.
* `requires-python = ">=3.10"` with no classifiers.

## 5. Detailed Changes
### 5.1 Create `tiferet_testing` Package
**Location:** `tiferet_testing/` at repo root.
**Files to create:**
* `tiferet_testing/pyproject.toml` — Separate build config:
    * `name = "tiferet-testing"`
    * `version` — dynamic, read from `tiferet_testing.__version__`
    * `dependencies = ["tiferet>=2.0.0b4", "pytest>=8.3.3"]`
    * `requires-python = ">=3.10"`
    * Classifiers for 3.10, 3.11, 3.12
* `tiferet_testing/tiferet_testing/__init__.py` — Public exports with `__all__`:
    * `MapperAssertions`, `AggregateTestBase`, `TransferObjectTestBase`
    * `DomainEventTestBase`, `ServiceEventTestBase`
    * `register_mapper_hooks`, `register_event_hooks`
    * `__version__ = '2.0.0b4'`
* `tiferet_testing/tiferet_testing/mappers.py` — Moved from `tiferet/mappers/tests/settings.py`. Change relative imports to absolute (`from tiferet.assets import TiferetError`, `from tiferet.mappers import Aggregate, TransferObject`).
* `tiferet_testing/tiferet_testing/events.py` — Moved from `tiferet/events/tests/settings.py`. Change relative imports to absolute.
* `tiferet_testing/tiferet_testing/hooks.py` — Extracted from the two `conftest.py` files. Provides reusable `register_mapper_hooks(metafunc)` and `register_event_hooks(metafunc)` functions that consumers call from their own `conftest.py`.
**Consumer usage:**
```python
# Consumer's conftest.py
from tiferet_testing.hooks import register_mapper_hooks, register_event_hooks
def pytest_generate_tests(metafunc):
    register_mapper_hooks(metafunc)
    register_event_hooks(metafunc)
```
```python
# Consumer's test file
from tiferet_testing import AggregateTestBase
class TestMyAggregate(AggregateTestBase):
    aggregate_cls = MyAggregate
    sample_data = {...}
    equality_fields = [...]
    set_attribute_params = [...]
```
### 5.2 Relocate Framework Tests
**Move:** All `tiferet/<module>/tests/` directories → `tests/<module>/`.
**Move:** `tiferet/tests_int/` → `tests_int/`.
Mapping (9 unit test suites + 1 integration suite):
* `tiferet/blueprints/tests/` → `tests/blueprints/`
* `tiferet/contexts/tests/` → `tests/contexts/`
* `tiferet/di/tests/` → `tests/di/`
* `tiferet/domain/tests/` → `tests/domain/`
* `tiferet/events/tests/` → `tests/events/` (minus `settings.py` and `conftest.py` which move to `tiferet_testing`)
* `tiferet/interfaces/tests/` → `tests/interfaces/`
* `tiferet/mappers/tests/` → `tests/mappers/` (minus `settings.py` and `conftest.py`)
* `tiferet/repos/tests/` → `tests/repos/`
* `tiferet/utils/tests/` → `tests/utils/`
* `tiferet/tests_int/` → `tests_int/`
**Import rewrite:** All test files change relative imports (`from ...assets import TiferetError`) to absolute imports (`from tiferet.assets import TiferetError`). Harness imports change to `from tiferet_testing import AggregateTestBase`, etc.
**New conftest.py files:**
* `tests/conftest.py` — Root conftest (shared fixtures if any).
* `tests/mappers/conftest.py` — Calls `register_mapper_hooks`.
* `tests/events/conftest.py` — Calls `register_event_hooks`.
### 5.3 Add `__all__` to All `__init__.py`
Add explicit `__all__` lists to control the public API. Applies to every `__init__.py` in the `tiferet` package and the `tiferet_testing` package.
Packages to update (10 files):
* `tiferet/__init__.py`
* `tiferet/assets/__init__.py`
* `tiferet/blueprints/__init__.py`
* `tiferet/contexts/__init__.py`
* `tiferet/di/__init__.py`
* `tiferet/domain/__init__.py`
* `tiferet/events/__init__.py`
* `tiferet/interfaces/__init__.py`
* `tiferet/mappers/__init__.py`
* `tiferet/utils/__init__.py`
* `tiferet_testing/tiferet_testing/__init__.py` (new, created with `__all__` from the start)
Pattern:
```python
"""Tiferet Domain Exports"""
# *** exports
__all__ = [
    'DomainObject',
    'AppInterface',
    'AppServiceDependency',
    ...
]
# ** app
from .settings import DomainObject
...
```
### 5.4 Update Core `pyproject.toml`
* Bump `__version__` → `'2.0.0b4'` in `tiferet/__init__.py`.
* Add Python version classifiers for 3.10, 3.11, 3.12.
* Update `requires-python = ">=3.10"` (no ceiling — already supports 3.12).
* Remove `tests/` subdirectories from `[tool.setuptools.packages.find]` includes (they're being moved out).
* Add `tiferet-testing` as a test optional dependency: `test = ["tiferet-testing>=2.0.0b4", "pytest>=8.3.3", "pytest_env>=1.1.5"]`.
* Add `[tool.pytest.ini_options]` section: `testpaths = ["tests", "tests_int"]`.
### 5.5 Shared Version Management
To mirror `Directory.Build.props` from .NET, the version is kept in sync manually between `tiferet/__init__.py` and `tiferet_testing/tiferet_testing/__init__.py`. Both declare `__version__ = '2.0.0b4'`. The `tiferet-testing` package pins `tiferet>=2.0.0b4` as a dependency, ensuring version compatibility.
## 6. Migration Impact
### For framework contributors
* Run tests with `pytest tests/ tests_int/` from repo root (or configure via `pyproject.toml` testpaths).
* Install dev dependencies: `pip install -e . -e tiferet_testing/[test]` (editable installs of both packages).
### For downstream consumers of harnesses
* **Before:** `from tiferet.mappers.tests.settings import AggregateTestBase` (deep internal import, fragile).
* **After:** `pip install tiferet-testing` then `from tiferet_testing import AggregateTestBase` (stable public API).
### For downstream consumers of the framework
* No breaking changes to the `tiferet` public API.
* Published package is lighter (no test files bundled).
* `__all__` provides explicit API surface; `from tiferet.domain import *` now only imports declared symbols.
## 7. Acceptance Criteria
1. `tiferet_testing/` package exists with its own `pyproject.toml` and is independently buildable (`pip install -e tiferet_testing/`).
2. All harness classes (`AggregateTestBase`, `TransferObjectTestBase`, `DomainEventTestBase`, `ServiceEventTestBase`, `MapperAssertions`) are importable from `tiferet_testing`.
3. No `tests/` directories remain inside `tiferet/`; all test files live under the top-level `tests/` and `tests_int/` directories.
4. `pytest tests/` passes with all existing tests (import paths updated).
5. `pytest tests_int/` passes for integration tests.
6. Every `__init__.py` in `tiferet/` and `tiferet_testing/` has an `__all__` list.
7. `python -c "from tiferet.domain import *; print(dir())"` only shows `__all__`-declared names.
8. `pip install tiferet` does not include test files in the installed package.
9. Python version classifiers for 3.10, 3.11, 3.12 are present in both `pyproject.toml` files.
